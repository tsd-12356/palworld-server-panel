# v3.0.0

发布日期：2026-07-14

v3.0 聚焦 Palworld 1.0 环境下的在线玩家准确性、RCON 稳定性和内部 REST API 安全边界。

## 主要改进

### REST 优先的在线玩家列表

- 面板优先调用 Palworld REST API 获取结构化在线玩家数据。
- 在线玩家列表可显示玩家名称、账号或 Player ID、等级和 Ping；默认不向浏览器暴露玩家 IP。
- REST 返回空列表时会准确显示 `0 人`，不会误判为 RCON 失败。
- REST 不可用时，面板会在已开启玩家列表的情况下回退到 RCON `ShowPlayers`。
- 玩家查询结果会短暂缓存，减少面板定时刷新对游戏服务端的压力。
- 页面会显示数据来源：`REST API`、`RCON 回退` 或 `不可用`。

### RCON 可靠性与配置保护

- 改进 Palworld RCON 响应读取，兼容服务端不返回传统 Source RCON 完成哨兵的行为。
- 面板与后台更新脚本共用 RCON 实现，避免两套逻辑发生偏差。
- `AdminPassword`、`RCONPort`、`RCONEnabled`、游戏与查询端口等由部署配置管理的字段，不能再通过通用配置表单修改。
- 配置接口不再返回上述敏感或部署管理字段，避免管理员密码与面板 RCON 凭据漂移。

### Docker 内部 REST API

Docker Compose 可通过以下 `.env` 配置启用 REST：

```env
PALWORLD_REST_ENABLED=true
PALWORLD_REST_PORT=8212
PALWORLD_REST_PATH=/v1/api/players
PALWORLD_REST_USERNAME=admin
# 留空时复用 RCON_PASSWORD
PALWORLD_REST_PASSWORD=
```

启用或修改 REST 设置后，需要重启游戏容器：

```bash
docker compose restart palworld
```

`8212/tcp` 仅由 Compose 内部网络中的面板容器访问，默认**不会**发布到宿主机或公网。不要在 `docker-compose.yml` 中添加 `8212:8212`，除非你已额外配置 TLS、访问控制和防火墙规则。

## 升级说明

1. 先备份 `data/palworld`，至少保留当前的 `PalWorldSettings.ini` 与世界存档。
2. 拉取 v3.0 代码并重建面板与游戏镜像：

   ```bash
   git pull
   docker compose up -d --build
   ```

3. 如果需要准确在线玩家数，在 `.env` 中设置 `PALWORLD_REST_ENABLED=true`，然后执行：

   ```bash
   docker compose restart palworld
   ```

   此操作会使在线玩家短暂掉线。

4. 检查面板“在线玩家”页面是否显示 `REST API` 来源；确认 `docker compose ps` 没有发布 `8212/tcp` 到宿主机。

## 验证

新增 REST 玩家查询、RCON 响应处理和部署管理配置保护测试。Docker 面板镜像构建期间会执行测试套件。

## 已知限制

- REST API 的路径和 Basic Auth 用户名可能随 Palworld Dedicated Server 版本变化；可使用 `PALWORLD_REST_PATH` 和 `PALWORLD_REST_USERNAME` 覆盖默认值。
- RCON 仍保留为兼容回退和控制台管理通道，但新版本 Palworld 可能不返回 `ShowPlayers` 的文本结果。
- MOD 管理仍是实验性功能；请在应用 MOD 前备份存档并参考 MOD 作者的兼容性说明。
