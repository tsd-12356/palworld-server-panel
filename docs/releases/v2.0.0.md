# v2.0.0

Palworld Server Panel 2.0 重点补齐 MOD 管理，同时继续保留 Docker Compose 一键部署、原生 systemd 安装器、存档管理、配置管理、手动更新和浅色毛玻璃 UI。

## Highlights

- 新增 `MOD 管理` 标签页。
- 支持上传 `.pak`、`.sig`、`.zip` MOD 文件。
- 支持启用、禁用、移入废纸篓和清理废纸篓。
- 支持识别官方 `Info.json` 包，并在启用前提示 Linux/Docker 兼容风险。
- 支持“应用并重启”：先尝试备份当前存档，再重启 Palworld 让 MOD 变更生效。
- Docker Compose 和 Ubuntu/Debian systemd 两种部署方式继续保留。
- README 和 MOD 文档补充了安装路径、回滚方式、安全边界和客户端兼容提示。

## Experimental MOD Notice

MOD 管理在 2.0 中是实验性功能。面板负责安全上传、文件移动、启用状态维护、废纸篓和重启流程，但不保证任何第三方 MOD 一定能在你的 Palworld 版本、Linux/systemd 或 Docker 服务端环境中正常运行。

建议：

- 安装前先备份存档。
- 一次只测试一个 MOD。
- 按 MOD 作者说明确认服务端和客户端是否都需要安装。
- 出现无法启动或无法进服时，先禁用最近添加的 MOD，再查看实时日志。

## Upgrade Notes

Docker 用户：

```bash
git pull
cp .env.example .env  # 已有 .env 不要直接覆盖，只按需补新变量
docker compose up -d --build
```

systemd 用户：

```bash
git pull
sudo bash install.sh
sudo systemctl restart palworld-panel.service
```

2.0 新增的 MOD 相关环境变量：

```env
MOD_LIBRARY_ROOT=/app/data/mod-library
MOD_UPLOAD_MAX_BYTES=1073741824
PALWORLD_MOD_MODE=pak-safe
```

原生 systemd 部署中默认路径为 `/home/demo/palworld-panel/mod-library`。
